# Security Policy

感谢你帮助提升项目安全性。

## 请不要公开提交这些内容

- 服务器密码
- 真实 API Key / Token / Secret
- 生产数据库
- 真实 `.env`
- 私有部署恢复资料

## 漏洞披露方式

如果你发现的是安全问题，而不是普通功能缺陷，请不要直接公开发 Issue。

推荐做法：

1. 通过 GitHub 账号 `@86Kolton` 联系维护者
2. 或使用 GitHub 私有安全披露方式进行说明
3. 描述复现步骤、影响范围、可能风险和建议修复方向

## 报告中建议包含

- 漏洞类型
- 影响模块
- 复现条件
- 利用路径
- 可能影响的数据或权限
- 修复建议

## 处理原则

- 尽量先私下确认，再公开说明
- 修复前不公开敏感细节
- 修复后可补充公开说明与致谢
